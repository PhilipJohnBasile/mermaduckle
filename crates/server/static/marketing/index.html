Governance

Human review, audit depth, and recovery paths are part of the story.

Mermaduckle is not just a prompt launcher. The app exposes pending approvals, stores audit events, generates workflow reports, and keeps the surrounding operating data that teams need when automation touches production systems.

Approval queue

Pause executions at `pending_approval` and resume them with an explicit action.

Audit timeline

Track workflow, agent, and approval events with severity and actor metadata.

Executive report endpoint

Generate per-workflow audit summaries when stakeholders need a readable artifact.

Deployment

Fly-ready by default.

The repo already includes a dedicated Fly Dockerfile, a `fly.toml`, and deployment notes. The site now treats deployment as a first-class part of the product instead of an afterthought, and the repo can deploy from GitHub Actions on push to `main`.

Deploy command Full guide 
flyctl deploy -a <app-name> --dockerfile deploy/Dockerfile.fly

Use the local helper script or let GitHub Actions run this remotely on Fly.

Included in the repo

  • deploy/Dockerfile.fly for multi-stage Rust builds
  • fly.toml with the app name and internal service port
  • docs/deploy/fly.md covering domains, certs, and status checks
  • .github/workflows/fly-deploy.yml for push-to-main deployment
